ADR-005 高危操作必须 Vant Dialog 三步走
决策删除、注销、退出等不可逆操作，必须走 JS 调用 + JSON 声明 + WXML 挂载三段式
原因wx.showModal 样式不可定制，无品牌统一性；Vant Dialog 的 await Dialog.confirm() 提供清晰的 async/await 流程控制
影响每新增高危操作页面必须挂载 <van-dialog id="van-dialog" />，JSON 声明 Vant 组件
ADR-002 路由统一使用 app.navTo
决策所有页面跳转必须通过 app.navTo()，禁止直接调用 wx.navigateTo / wx.navigateBack
原因app.navTo 封装了参数序列化、登录态校验和全局拦截逻辑。直接调用会绕过这些保护
影响跳转时参数 key 必须与目标页 onLoad(options) 解构一致
